## 1. 컴포넌트의 목적

모든 페이지 상단에 고정되어 **브랜드 아이덴티티(발주처)** 를 표시하고,
**전역 기능(자산 검색 · 알림 · 도움말 · 사용자)** 의 진입점을 제공하는 영역.

## 2. 현재 구현 (As-Is)

### 구성 (좌 → 우)

```
┌──────────────────────────────────────────────────────────────┐
│ Samsung C&T   [spacer]   [🔍 Search Assets...]  🔔(●) ❓ (JH) │  64px 고정
└──────────────────────────────────────────────────────────────┘
```

| 요소 | 구현 상태 | 동작 여부 |
|---|---|---|
| 브랜드 "Samsung C&T" | 정적 텍스트 (`ui.py` `render_topbar`) | 표시만 (정상) |
| 검색창 "Search Assets..." | 순수 HTML `<input>` | **미동작** — 입력해도 아무 일도 안 일어남 |
| 알림 벨 + 빨간 점 | 정적 SVG + CSS 점 | **미동작** — 점이 항상 켜져 있음 (가짜 신호) |
| 도움말 (?) | 정적 SVG | **미동작** |
| 아바타 "JH" | 하드코딩 이니셜 | **미동작** — 사용자 개념 없음 |

- 렌더링: `position: fixed` + `z-index: 9000`, 사이드바/본문을 64px 아래로 밀어냄
- `render_topbar(active_page)`가 인자를 받지만 **사용하지 않음** (내비게이션 탭이 없는 구조)

### 문제점

| # | 문제 | 상세 |
|---|---|---|
| P1 | **우측 기능 4종이 전부 장식** | 검색·알림·도움말·아바타가 클릭/입력에 무반응. 사용자가 동작을 기대하게 만드는 UI 부채 |
| P2 | **알림 빨간 점이 가짜** | 데이터와 무관하게 항상 켜져 있음 — "조치 대기" 등 실제 알림과 연결 안 됨 |
| P3 | **Streamlit 네이티브 헤더 노출** | 우상단에 Deploy 버튼·메뉴(⋮)가 커스텀 상단바와 겹쳐 표시됨. 데모/운영 화면에 노출 부적절 |
| P4 | 죽은 CSS | `.ps-topbar-nav` 스타일(50~61행)이 정의만 되고 HTML에서 미사용 |
| P5 | `active_page` 미사용 인자 | `render_topbar(st.session_state["page"])` 호출이 무의미 |

## 3. 목표 (To-Be)

> **상단바의 모든 요소가 "보이는 대로 동작"하거나, 동작하지 않는 것은 제거한다.**

### 요구사항

| ID | 요구사항 | 우선순위 |
|---|---|---|
| R1 | **알림 실데이터 연동** — 빨간 점·배지를 "조치 대기 통보서 수 + 지연 태스크 수"로 계산. 0이면 점 숨김, 클릭 시 지적·오동작 관리(조치 대기 필터)로 이동 | P0 |
| R2 | **네이티브 헤더 정리** — `stAppHeader`(Deploy·메뉴)를 CSS로 숨겨 커스텀 상단바와의 중첩 제거 | P0 |
| R3 | **검색 동작화** — 검색어 입력 후 Enter 시 시설 관리 페이지로 이동해 장비명/위치ID 필터 적용 (`?search=` query param 경유). v1에서 과하면 검색창 제거가 차선 | P1 |
| R4 | **아바타 실사용자화** — 로그인 도입(v2) 전까지는 설정값(secrets `USER_INITIALS` 또는 기본 점검자명)으로 표시 | P2 |
| R5 | **도움말 연결** — 사용 가이드 문서/모달로 연결하거나 제거 | P2 |
| R6 | 죽은 코드 정리 — `.ps-topbar-nav` CSS 제거, `render_topbar()` 미사용 인자 정리 | P2 |

### 비요구사항

- 사용자 인증/세션 — [PRD.md](PRD.md) v2 거버넌스 과제. 아바타는 그때 실계정과 연결
- 전역 검색(점검·지적·통보서 통합 검색) — v1은 장비 검색만
- 다국어/반응형 모바일 최적화 — 별도 과제

## 4. 구현 계획

1. **R2 (네이티브 헤더 숨김)** — `THEME_CSS`에 추가:
   ```css
   header[data-testid="stHeader"] { display: none !important; }
   ```
   ※ 로컬 개발 시 Deploy 버튼이 사라져도 무방 (배포는 git push 기반)
2. **R1 (알림 연동)** — `render_topbar()`에서 `data.load_notices()`·`load_tasks()`로
   대기 건수 계산 → 0이면 `ps-dot` 미출력, 1+이면 점(또는 숫자 배지) 출력.
   벨을 `<a href="?page=deficiencies">` 형태로 감싸 클릭 이동 지원
3. **R3 (검색)** — `<input>`을 `<form>`으로 감싸 GET 제출 → `app.py`에서
   `st.query_params["search"]` 감지 시 시설 관리로 라우팅 + 필터 초기값 주입
4. **R4~R6** — 소규모 정리 (각 1~5줄)
5. **검증** — ① Deploy 버튼 미노출 ② 조치 대기 0건 세션에서 점 숨김 ③ 점검 불량
   입력 → 점 표시 ④ 검색어 제출 → 시설 관리 필터 적용

## 5. 영향 범위 및 리스크

- `lib/ui.py`에서 `lib/data` import 추가 (현재 미참조) — `data.py`는 `ui`를 참조하지
  않으므로 순환 import 없음
- 상단바가 데이터를 읽게 되면 **매 rerun마다 호출**되므로 계산은 카운트 수준으로 제한
- R3의 query param 라우팅은 기존 QR 딥링크(`?eq=`) 처리와 같은 위치(`app.py`)에서
  분기 — 딥링크 우선순위 유지 필요 (`eq` > `search`)
- 데모 관점: P1~P2만 해결해도 "장식 UI" 인상이 크게 줄어듦 — R1·R2를 1차 배포로,
  R3 이하는 후속 배포로 분리 권장
